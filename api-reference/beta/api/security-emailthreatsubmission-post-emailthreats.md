# <a name="create-emailthreatsubmission"></a>Criar emailThreatSubmission
Namespace: microsoft.graph.security

[!INCLUDE [beta-disclaimer](../../includes/beta-disclaimer.md)]

Crie um novo [objeto emailThreatSubmission](../resources/security-emailthreatsubmission.md) .

## <a name="permissions"></a>Permissões
Uma das seguintes permissões é obrigatória para chamar esta API. Para saber mais, incluindo como escolher permissões, confira [Permissões](/graph/permissions-reference).

|Tipo de permissão|Permissões (da com menos para a com mais privilégios)|
|:---|:---|
|Delegado (conta corporativa ou de estudante)|ThreatSubmission.ReadWrite, ThreatSubmission.ReadWrite.All|
|Delegado (conta pessoal da Microsoft)|Sem suporte|
|Aplicativo|ThreatSubmission.ReadWrite.All|

## <a name="http-request"></a>Solicitação HTTP

<!-- {
  "blockType": "ignored"
}
-->
``` http
POST /security/threatSubmission/emailThreats
```

## <a name="request-headers"></a>Cabeçalhos de solicitação
|Nome|Descrição|
|:---|:---|
|Autorização|{token} de portador. Obrigatório.|
|Content-Type|application/json. Obrigatório.|

## <a name="request-body"></a>Corpo da solicitação
No corpo da solicitação, forneça uma representação JSON de um objeto [emailContentThreatSubmission](../resources/security-emailcontentthreatsubmission.md) ou um [objeto emailUrlThreatSubmission](../resources/security-emailurlthreatsubmission.md) .

Os **recursos emailContentThreatSubmission** e **emailUrlThreatSubmission** são ambos subtipos da entidade **emailThreatSubmission** . Escolha o que incluir no corpo da solicitação com base no seguinte:  
* Se você quiser criar um envio com o próprio conteúdo de email, inclua um **objeto emailContentThreatSubmission** .
* Se você quiser criar um envio com uma URL que aponte para o email, inclua um **objeto emailUrlThreatSubmission** .

Depois que os **objetos emailContentThreatSubmission** ou **emailUrlThreatSubmision** são criados, o serviço de envio de ameaças armazena apenas alguns metadados sobre o email. O conteúdo do email não é armazenado. Como resultado, uma **entidade emailThreatSubmission** é criada.

## <a name="response"></a>Resposta

Se tiver êxito, este método retornará um código `201 Created` de resposta e um [objeto emailThreatSubmission](../resources/security-emailthreatsubmission.md) no corpo da resposta.
